01 — About

Quality is my discipline.

More than test execution — I design automation systems that scale, fail gracefully, and report with clarity.

I am a QA Engineer and Automation Developer who works at the crossover between quality assurance and software engineering. My work is not confined to writing test scripts — it extends to designing framework architecture, building reusable automation infrastructure, and developing Python tooling that eliminates manual overhead across entire teams.

At Schneider Electric, I work on industrial-grade desktop software used by electrical engineers worldwide. This demands precision: automating WPF applications with non-standard controls, mapping complex object trees, and maintaining regression suites that cover topology exports and configuration workflows.

"Breaking software before users do — methodically, repeatedly, and at scale."

Outside enterprise work, I pursue personal automation projects — from Playwright-based cross-platform frameworks to Python bots and scraping utilities. Great automation engineers think like developers first.

Currently deepening expertise in advanced Playwright patterns, API testing at scale, and CI/CD pipeline integration — always moving the craft forward.
